# Criterios de aceptación

- Proyecto Angular aislado en `frontend-estudiante-angular/app`.
- Backend existente sin cambios funcionales; el frontend se conecta por la API REST documentada.
- Compilación ejecutable con `npm run build`.
- Flujo estudiante completo conectado a datos reales del backend con respaldo mock visual.
- Operaciones importantes resueltas dentro de la interfaz, sin alertas nativas.
- Consentimiento requerido antes de iniciar una simulación.
- Decisiones almacenadas en estado local con feedback y XP.
- Bitácora requerida para el cierre.
- 46 mockups fuente disponibles para QA en `/student/reference`.
- Estética basada en los mockups: fondo oscuro, retícula, paneles técnicos, cian, verde, ámbar y animaciones discretas.
